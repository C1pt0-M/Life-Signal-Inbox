# 开发过程记录

## 2026-04-19 OCR 接入

本次把截图文字识别从模拟结果改成了本地 `PaddleOCR`（本地文字识别库）。

### 主要改动

- `backend/app/ocr.py`（文字识别适配器）改为真实本地识别。
- 使用懒加载：只有用户上传图片时才初始化 `PaddleOCR`（本地文字识别库），避免后端启动变慢。
- 上传图片会先写入临时文件，识别完成后立即删除。
- 识别结果统一整理为：
  - `text`（识别出的完整文本）
  - `confidence`（平均可信度）
  - `blocks`（逐行文本块）
  - `provider`（识别来源）
- 保留失败兜底：库未安装或运行失败时，接口返回错误信息，不影响整个后端启动。

### 版本选择

实测 `paddleocr 3.4.1`（文字识别库）搭配 `paddlepaddle 3.3.1`（运行框架）在 Windows CPU 环境会触发：

```text
ConvertPirAttribute2RuntimeAttribute not support [pir::ArrayAttribute<pir::DoubleAttribute>]
```

改为 `paddlepaddle 3.2.0`（运行框架）后，本地截图 OCR 可以正常运行。

### 当前默认配置

默认使用轻量模型：

```text
PP-OCRv5_mobile_det
PP-OCRv5_mobile_rec
```

并关闭：

```text
doc orientation classify
doc unwarping
textline orientation
```

原因是本项目主要处理微信截图、课程通知截图、社区公告截图，不需要复杂文档矫正，优先保证演示稳定和响应速度。

### 已验证

- 单元测试覆盖结果解析、临时文件清理、库缺失兜底。
- 使用用户提供的任务截图实测，`PaddleOCR`（本地文字识别库）可以识别出中文正文，平均可信度约 `0.93`。

### 后续可优化

- 增加图片预处理，例如放大、灰度化、锐化。
- 对竖排文字、倾斜截图开启方向分类。
- 把模型路径固定到项目配置，避免不同机器重复下载。

## 2026-04-19 四象限拖拽调整

本次把四象限规划从只读展示改成了可手动调整。

### 主要改动

- 四象限页的事项卡片支持拖动。
- 用户可以把事项拖到任意象限。
- 调整结果会写入 `localStorage`（浏览器本地存储），刷新页面后仍然保留。
- 新增纯函数测试，覆盖用户自定义象限覆盖逻辑。

### 已验证

- 前端单元测试通过。
- 前端构建通过。
- 使用浏览器自动化实测：从“重要且紧急”拖到“不重要且不紧急”后，卡片成功移动，没有控制台错误。
